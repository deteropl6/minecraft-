Данный датапак полностью изменяет все торговые сделки у жителей. Теперь основная валюта для торговли не изумруды, а железные и золотые монеты. Железные и золотые монеты можно получить только путем торговли с жителями. Вы можете так же обменять железные монеты на золотые и обратно у священника и библиотекаря на уровне "эксперта". Добавлено более 500 новых торговых сделок у всех жителей. 

Лучше всего устанавливать данный датапак на новый мир, где еще жители не определились со своей профессией. Если житель уже выбрал профессию, то изменить ее он уже не сможет. 

Вы можете так же изменить торговые сделки у любого жителя. Вся подробная информация по редактированию торговых сделок указана в файле "Основная информация", который расположен в самом датапаке. 

Данный датапак "Advanced Villager Trades" сделан по структуре датапака "Custom Villager Trades".
Автор датапака: Dragon_XY.
-------------------------------------
Торговые сделки жителей содержатся в отдельных файлах, которые расположены в папке с названием профессии жителя, которые расположены в папке data/cv/functions/default. В каждой папке содержится 5 файлов, каждый из которых отвечает за торговые сделки на каждом уровне профессии (от 1 до 5, где 1 уровень - новичок, а 5 уровень - мастер). Файлы формата .mcfunction можно открыть с помощью любого текстового редкатора, даже с помощью обычного блокнота. 
-------------------------------------
execute as @e[type=minecraft:villager,tag=!check1,nbt={VillagerData:{profession:"minecraft:PROFESSION", level:1}},limit=1,sort=nearest] run data remove entity @e[type=minecraft:villager,tag=!check1,nbt={VillagerData:{profession:"minecraft:PROFESSION", level:1}},limit=1,sort=nearest] Offers.Recipes[-1]
execute as @e[type=minecraft:villager,tag=!check1,nbt={VillagerData:{profession:"minecraft:PROFESSION", level:1}},limit=1,sort=nearest] run data remove entity @e[type=minecraft:villager,tag=!check1,nbt={VillagerData:{profession:"minecraft:PROFESSION", level:1}},limit=1,sort=nearest] Offers.Recipes[-1]
-------------------------------------
Эти команды, описанные выше, удаляют стандартные торговые сделки жителей. Каждая команда удаляет 1 торговую сделку. Если вы хотите сохранить стандартные торговые сделки жителей и просто добавить дополнительные, то удалите эти строки.
-------------------------------------
execute as @e[type=minecraft:villager,tag=!check1,nbt={VillagerData:{profession:"minecraft:PROFESSION", level:1}},limit=1,sort=nearest] store result score @s RNG1 run loot spawn ~ -2 ~ loot cv:1
-------------------------------------
Эта команда, описанная выше, выбирает случайное число, указанное в cv:N, где N - число ваших сделок на выбор. Возможное количество сделок - от 1 до 50. Например, изменение cv:1 на cv:10 означает, что у вас есть 10 сделок на выбор. Если у вас всего 3 торговых сделки, а вы поставили cv:5, то есть шанс, что торговых сделок у жителя вообще не будет, так как случайное число может выпасть на 4 или на 5, где ничего не указано.  
Что бы сгенерировать команду для выборки торговых сделок из другого списка, нужно просто изменить RNG1 на RNG2 и так далее, максимум до RNG10.
В данной команде RNG указывает на количество торговых сделок, которые отображаются у жителя. Максимум может быть RNG10, то есть 10 торговых сделок каждый уровень торговца. 
-------------------------------------
execute if score @e[type=minecraft:villager,tag=!check1,nbt={VillagerData:{profession:"minecraft:PROFESSION", level:1}},limit=1,sort=nearest] RNG1 matches 1 run data modify entity @e[type=minecraft:villager,tag=!check1,nbt={VillagerData:{profession:"minecraft:PROFESSION", level:1}},limit=1,sort=nearest] Offers.Recipes append value {buyB:{id:"minecraft:air",Count:1},buy:{id:"minecraft:ITEMID",Count:###},sell:{id:"minecraft:ITEMID",Count:###},xp:###,rewardExp:1b,maxUses:###}
-------------------------------------
Эта команда, описанная выше, генерирует сделку. Добавьте по одной такой команде для каждой сделки на выбор. Например 5 сделок - это 5 исполнений, а cv:5.
RNG1: Первая сделка для генерации.
matches 1: Для первой сделки на выбор, matches 2 - для второй сделки и так далее, максимум до matches 50. Количество matches равно количеству случайных сделок на выбор - cv.
Так же каждая случайная сделка относится к определенному номеру торговой сделки у жителя - RNG. Например, все сделки с RNG1 будут относится к торговой сделке 1, с RNG2 - к торговой сделке 2.
Например, у вас 3 сделки с одинаковыми RNG1 и matches 1, то при выполнении команды, все сделки с таким названием будут у торговца в количестве 3 штук.
Например, у вас 3 сделки с одинаковыми RNG1 и разными matches (1, 2 и 3), то при выполнении команды, у торговца будет только одна сделка из них, которая будет выбрана случайным образом. 
Для других сделок в данной команде необходимо просто изменить RNG1 на RNG2 и так далее, максимум до RNG10.
-------------------------------------
execute as @e[type=minecraft:villager,tag=!check1,nbt={VillagerData:{profession:"minecraft:PROFESSION", level:1}},limit=1,sort=nearest] run data modify entity @s Xp set value 0
execute as @e[type=minecraft:villager,tag=!check1,nbt={VillagerData:{profession:"minecraft:PROFESSION", level:1}},limit=1,sort=nearest] run tag @s add check1
scoreboard players reset @e
-------------------------------------
Для команд, описанных выше:
Для первой команды: Если вы хотите, что бы житель любой профессии на уровне "новичок" мог ее изменить, то установите set value 0. Если вы хотите, что бы житель не мог изменить уже выбранную профессию, то установите - set value 1. Если житель уже выбрал профессию и он находится на уровне новичка, то при уборке его рабочего места, торги у этого жителя сбрасываются. Если поставить снова любое рабочее место, тогда он заново выберет профессию.
Для второй команды: Команда устанавливает метку о том, что торговые сделки жителя были изменены для его текущего уровня. Это нужно для того, чтобы торговые сделки не повторялись каждый тик.
-------------------------------------
{buyB:{id:"minecraft:air",Count:1},buy:{id:"minecraft:ITEMID",Count:###},sell:{id:"minecraft:ITEMID",Count:###},xp:###,rewardExp:1b,maxUses:###}
-------------------------------------
Эта команда, описанная выше, определяет детали торговой сделки.

buy: Это то, что покупает житель. 
buyB: Это то, что покупает житель. Используется как дополнительный слот для торговли. По умолчанию он пуст (minecraft:air).
sell: Это то, что продает житель. 

ITEMID: Здесь указывается ИД предмета. Можно его сделать пустым, заменив ITEMID на air.
Count ###: Измените ### на количество каждого товара, которое покупается/продается.
"minecraft:air": Если вам нужен дополнительный слот для торговли, измените "air" на ИД необходимого вам предмета.
"minecraft": Если вы хотите торговать предметами из модов, измените "minecraft" на имя мода, из которого вы хотите добавить предмет. Нажмите F3+H в игре для получения дополнительных всплывающих подсказок. Это даст вам дополнительную информацию о предмете при наведении курсора мыши.

xp: Это то, сколько опыта получает житель за торговлю. Измените ###, чтобы установить количество опыта.
rewardExp: Если установлено значение 1b(true), вы получаете такое же количество опыта. Установите значение 0b(false), чтобы не получать опыта от торговли.
maxUses: Измените ###, чтобы установить количество торговых сделок между пополнениями запасов у жителя. 
-------------------------------------
Для продажи/покупки вы также можете добавлять чары.
Пример зачарованной книги:
{id:enchanted_book,Count:1,tag:{StoredEnchantments:[{id:"smite",lvl:5}]}}

Пример зачарованного предмета:
{id:diamond_sword,Count:1,tag:{Enchantments:[{id:"looting",lvl:3}]}}
